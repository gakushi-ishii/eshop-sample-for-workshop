# 03. 検索結果の価格ソート／絞り込みを Coding Agent に任せる

このラボでは、要件と受け入れ条件を GitHub Issue にまとめ、Copilot Coding Agent に実装を委任します。
作業中に画面を見続けるのではなく、完成した Pull Request と CI の結果を後から確認します。

## なぜ Coding Agent を使うのか

価格ソートと価格帯フィルタの中心は純粋なデータ処理であり、期待結果をテストで表現できます。
要件を Issue に書いて非同期に任せることで、ローカル Agent Mode との使い分けを体験できます。

> **判断ルール:** 受け入れ条件をテストで表現できる作業は、Coding Agent への委任が向いています。

## 前提

- リポジトリで GitHub Issues と Copilot Coding Agent を利用できること。
- 参加者の PC で Node.js を起動する必要はありません。
- PR の CI（[`../.github/workflows/ci.yml`](../.github/workflows/ci.yml)）がテストとビルドを実行します。

## 1. Issue を作成し、Coding Agent に委任する

次のプロンプト内の要件を確認してから、Copilot App に貼り付けます。

### コピー用プロンプト

```text
Outdoor eShop に「検索結果の価格ソートと価格帯フィルタ」を追加する GitHub Issue を作成し、
その Issue を Copilot Coding Agent に割り当ててください。

Issue には次の内容を含めてください。

目的:
- 利用者が検索結果を価格の安い順または高い順に並べ替えられるようにする。
- 利用者が最低価格と最高価格を指定して商品を絞り込めるようにする。

受け入れ条件:
- 画面上で「標準」「価格の安い順」「価格の高い順」を選択できる。
- 最低価格、最高価格、または両方を指定して絞り込める。
- キーワード検索、価格帯フィルタ、並べ替えを同時に適用できる。
- 条件を解除すると全商品へ戻る。
- 元の商品配列を変更しない。
- 空の結果を画面上で分かりやすく示す。
- 検索、価格帯、並べ替えの主要な組み合わせを src/lib/search.test.ts でテストする。
- 既存テストを維持し、npm test と npm run build を成功させる。

実装上の注意:
- 既存の React、TypeScript、CSS の構成とデザインに合わせる。
- 新しい外部依存は、必要性を説明できる場合を除いて追加しない。
- 課題と無関係なリファクタリングは行わない。

Issue 作成後に、Issue 番号、登録した受け入れ条件、Coding Agent への割り当て状態を報告してください。
```

## 2. Coding Agent の進行状況を確認する

Coding Agent が作業を開始すると、Issue に進行状況が表示され、完了後に PR が作成されます。
急いで追加指示を重ねず、まず Issue の要件に沿った PR が作成されるのを待ちます。

PR が作成されたら、Copilot App に状況整理を依頼します。

### コピー用プロンプト

```text
先ほどの Issue に関連して作成された Pull Request を確認してください。
Issue の受け入れ条件ごとに、PR のどの変更とテストが対応しているか整理してください。
CI の各チェックが何を検証しているかと、現在の成功・失敗・実行中の状態も説明してください。
この時点では PR を変更、承認、マージしないでください。
```

## 3. CI の結果を確認する

このリポジトリの CI は PR に対して次を実行します。

| チェック | 確認内容 |
| --- | --- |
| `npm ci` | lock file に従って依存関係を再現できる |
| `npm test` | 既存テストと追加テストが成功する |
| `npm run build` | TypeScript の型チェックと本番ビルドが成功する |

すべて成功しても、画面の使いやすさまで保証されたわけではありません。
次の Lab 04 で差分と実際の画面を人が確認します。

## 完了条件

- 受け入れ条件を含む Issue が作成された。
- Issue が Copilot Coding Agent に割り当てられた。
- Issue に関連する PR が作成された。
- PR の変更内容と CI の状態を説明できる。
- PR はまだマージしていない。
