# 04. Coding Agent が作った PR をローカルで検証する

[Lab 03](./03-challenge-b-search-sort.md) で Coding Agent が作成した PR を、
マージする前に差分、テスト、ビルド、実際の画面で検証します。
CI が成功していても、要件どおりで使いやすいかを判断するのは人の役割です。

## 前提

- Lab 03 で作成された PR の番号が分かっていること。
- Node.js 20 と npm を利用できること。

## 1. PR を別セッションで開いて検証する

`<PR番号>` を実際の番号に置き換えて、Copilot App に貼り付けます。
別のプロジェクトセッションで開くことで、Lab 02 の作業ブランチと PR の変更が混ざることを防ぎます。

### コピー用プロンプト

```text
Pull Request #<PR番号> をマージせず、ローカルで検証してください。

次の順序で進めてください。
1. この PR を現在の作業とは別のプロジェクトセッションで開く。
2. 新しいセッションで Issue と PR の説明を読み、変更差分を受け入れ条件と対応付けて確認する。
3. 依存関係が未導入または lock file が変わっている場合は npm ci を実行する。
4. npm test と npm run build を実行する。
5. npm run dev で開発サーバーを起動し、Browser Canvas でアプリを開く。
6. キーワード検索、価格の安い順・高い順、最低価格・最高価格、条件の組み合わせ、条件解除、0件表示を確認する。
7. 受け入れ条件ごとに合格または不合格を報告し、差分上の懸念、テスト結果、画面確認結果をまとめる。

検証中はコードを変更せず、PR の承認やマージも行わないでください。
エラーや不合格がある場合は、再現手順と期待結果を具体的に報告してください。
```

## 2. 検証結果を判断する

次のすべてを満たせば承認候補です。

- Issue の受け入れ条件をすべて満たしている。
- CI、`npm test`、`npm run build` が成功している。
- Browser Canvas で主要な操作が意図どおりに動く。
- 差分に課題と無関係な変更や不要な依存追加がない。
- エラー時や0件時の表示が利用者に分かりやすい。

## 3-A. 修正が必要な場合

検証結果をもとに、再現可能な PR コメントを作成して投稿します。

### コピー用プロンプト

```text
Pull Request #<PR番号> の検証で見つかった次の問題について、
再現手順、実際の結果、期待する結果、関連する受け入れ条件を含むレビューコメントを作成し、PR に投稿してください。

問題:
<ここに検証で見つかった問題を書く>

解決方法を細かく指定しすぎず、Coding Agent が修正結果をテストで確認できる内容にしてください。
PR の承認やマージは行わないでください。
```

Coding Agent の修正後は、手順1の検証をもう一度実施します。

## 3-B. 問題がない場合

検証結果を確認したうえで、承認だけを依頼します。

### コピー用プロンプト

```text
Pull Request #<PR番号> は、Issue の受け入れ条件、CI、ローカルテスト、ビルド、Browser Canvas での動作確認をすべて通過しました。
確認した内容を短いレビューコメントにまとめて、この PR を承認してください。
マージは行わないでください。
```

承認後のマージは、リポジトリのルールとワークショップ講師の指示に従って人が判断します。

## 完了条件

- PR の差分と Issue の受け入れ条件を対応付けて確認した。
- テスト、ビルド、Browser Canvas で検証した。
- 問題があれば再現可能なコメントを投稿し、修正後に再検証した。
- 問題がなければ PR を承認した。
- PR を勝手にマージしていない。
