# 01. 現在のアプリを起動して確認する

このラボでは、GitHub Copilot App にリポジトリの調査からアプリの起動までを任せ、
**Browser Canvas** で現在の画面とユーザーフローを確認します。
コードはまだ変更しません。次のラボで実装する課題を、実際の操作から見つけることが目的です。

## このラボで体験すること

- Copilot App にリポジトリの構成と起動方法を調べてもらう。
- 開発サーバーを起動し、Browser Canvas でアプリを表示する。
- 商品一覧とキーワード検索を操作する。
- 商品カードから詳細を見られない現状を確認し、次の課題を発見する。

## 前提

- GitHub Copilot App でこのリポジトリを開いていること。
- Node.js 20 と npm が利用できること（バージョンは `../.node-version` で指定）。

## 1. Copilot App に調査・起動・画面確認を依頼する

次のプロンプトをそのまま Copilot App に貼り付けます。

### コピー用プロンプト

```text
このリポジトリのアプリケーションについて、実装前の状態を確認したいです。
次の順序で進めてください。

1. README、package.json、src 配下の主要ファイルを調べ、アプリの目的、技術構成、現在利用できる機能を簡潔に説明してください。
2. このラボではコードを変更しないでください。
3. 依存関係が未導入なら、package-lock.json に従って npm ci を実行してください。
4. npm run dev で開発サーバーを起動し、起動完了とアクセス先 URL を確認してください。開発サーバーは次のラボでも使うため、終了せずに実行したままにしてください。
5. アプリを Browser Canvas で開いてください。
6. Canvas 上で商品一覧が表示されることと、検索ボックスで「テント」を検索すると結果が絞り込まれることを確認してください。
7. 商品カードから商品詳細を開けるか確認してください。
8. 最後に、確認できた現在の機能、商品一覧に関係する主なファイル、次に追加するとよい機能を報告してください。

エラーが発生した場合は、回避せずに原因と解決方法を示してください。
```

## 2. Browser Canvas で確認する

Copilot App が `http://localhost:5173/` を開いたら、次を確認します。

1. 9件の商品がカード形式で表示される。
2. 検索ボックスへ `テント` と入力すると、表示商品が絞り込まれる。
3. 検索文字を消すと、全商品が再表示される。
4. 商品カードを選んでも、商品ごとの詳細ページへ移動できない。

4つ目が、[Lab 02](./02-challenge-a-product-detail.md) で解決する課題です。

## 完了条件

- Browser Canvas に現在のアプリが表示されている。
- 商品一覧と検索が正常に動作することを確認できた。
- 商品詳細ページが未実装であることを確認できた。
- Copilot App がコードを変更していない。
- 開発サーバーが起動したままになっている。

## 起動できない場合の予備手順

通常は上のプロンプトで npm 起動まで完了します。Node.js のローカル環境を利用できない場合だけ、
次のいずれかを使用してください。

### Docker Desktop

```bash
docker compose up
```

`http://localhost:5173/` を Browser Canvas で開きます。ソースコードはホスト側にあるため、
次のラボで Copilot App が行う変更も反映されます。

### Dev Container / Codespaces

`../.devcontainer/` を利用すると Node.js 20 の環境が作成され、依存関係も自動でインストールされます。
ポート `5173` が転送されたら、その URL を Browser Canvas で開きます。
