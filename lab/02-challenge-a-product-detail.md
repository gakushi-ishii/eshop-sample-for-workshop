# 02. 商品詳細ページを作る

[Lab 01](./01-explore-current-app.md) では、商品一覧と検索は使える一方で、
商品カードから詳しい情報を確認できないことが分かりました。
このラボでは Copilot App に商品詳細ページを実装してもらい、変更差分の Canvas と実際の画面を確認します。

## なぜローカル Agent Mode を使うのか

画面遷移やレイアウトは、テスト結果だけでは使いやすさを判断できません。
Copilot App で実装し、変更差分の Canvas を確認し、Browser Canvas で操作する短いフィードバックループを体験します。

> **判断ルール:** 目で見て確かめながら直したい作業は、ローカル Agent Mode が向いています。

## 前提

- Lab 01 で開発サーバーを起動していること。
- Browser Canvas で `http://localhost:5173/` を表示できること。

## 1. 商品詳細ページの実装を依頼する

### 受け入れ条件

- 商品カードを選ぶと `/product/:id` 形式の URL で詳細ページが開く。
- 詳細ページに大きな商品画像、商品名、説明、税込価格が表示される。
- 一覧へ戻るための分かりやすい導線がある。
- 一覧へ戻った後も商品検索が利用できる。
- 存在しない商品 ID では、利用者が一覧へ戻れる案内を表示する。
- 既存テストと本番ビルドが成功する。

### コピー用プロンプト

```text
Lab 01 で確認した Outdoor eShop に、商品詳細ページを追加してください。
実装前に既存の構成と関連ファイルを調べ、現在のデザインと実装パターンを尊重してください。

受け入れ条件:
- 商品カードを選ぶと /product/:id 形式の URL で詳細ページを開く。
- 詳細ページには、大きな商品画像、商品名、説明、税込価格を表示する。
- 商品一覧へ戻るための分かりやすい導線を用意する。
- 一覧へ戻った後も既存の商品検索を利用できる。
- 存在しない商品 ID の場合は、商品が見つからないことと一覧へ戻る導線を表示する。
- キーボード操作を含め、商品カードを操作可能な UI にする。
- 必要なテストを追加または更新し、npm test と npm run build を成功させる。

不要な機能追加や商品データの変更は行わないでください。
実装後は、変更したファイルと設計上の判断、テスト結果を簡潔に説明してください。
まだコミットや Push はしないでください。
```

## 2. 変更差分を Canvas で確認する

Copilot App の変更差分 Canvas を開き、次を確認します。

1. 変更が商品詳細ページとそのテストに必要な範囲へ収まっている。
2. 商品データや既存検索が意図せず変更されていない。
3. URL の処理、詳細画面、一覧へ戻る処理が実装されている。
4. テストや型チェックを回避する設定が追加されていない。

差分の説明も Copilot App に依頼できます。

### コピー用プロンプト

```text
現在の変更差分を Canvas で表示し、受け入れ条件ごとに、どのファイルのどの変更で満たしているか説明してください。
意図していない変更、既存機能への影響、テスト不足があれば指摘してください。
この時点では追加の編集、コミット、Push は行わないでください。
```

## 3. Browser Canvas で動作確認する

### コピー用プロンプト

```text
開発サーバーの状態を確認し、必要なら npm run dev で再起動してください。
Outdoor eShop を Browser Canvas で開き、次の操作を順番に確認してください。

1. 商品一覧から任意の商品カードを選ぶ。
2. URL が /product/:id に変わり、画像、商品名、説明、税込価格が表示される。
3. 一覧へ戻る導線で商品一覧へ戻る。
4. 検索ボックスで「テント」を検索し、既存の検索が引き続き動作する。
5. 存在しない商品 ID の URL を開き、案内と一覧へ戻る導線が表示される。

各確認結果を合格または不合格で報告し、不合格の場合だけ原因と修正案を示してください。
勝手にコミットや Push はしないでください。
```

## 4. 見た目や操作を調整する

Browser Canvas を見て修正したい点があれば、具体的な状態と期待結果を伝えます。

```text
商品詳細ページでは商品画像が目立ちすぎています。
デスクトップでは画像と商品情報を左右に配置し、モバイルでは縦に並ぶように調整してください。
既存の配色と余白のルールは維持し、変更後に Browser Canvas で両方のレイアウトを確認してください。
npm test と npm run build も再実行し、結果を報告してください。
まだコミットや Push はしないでください。
```

調整が完了したら差分をもう一度確認します。

## 5. 変更を保存する

すべての確認を通過したら、Lab 02 の変更だけをコミットしてリモートへ Push します。

### コピー用プロンプト

```text
商品詳細ページの最終差分を確認し、このラボと無関係な変更が含まれていないことを確認してください。
npm test と npm run build が成功していることを再確認したうえで、商品詳細ページの変更だけをコミットし、現在の作業ブランチをリモートへ Push してください。
コミットの概要、Push したブランチ名、テストとビルドの結果を報告してください。
Pull Request の作成やマージはまだ行わないでください。
```

## 完了条件

- すべての受け入れ条件を差分と Browser Canvas の両方で確認できた。
- `npm test` と `npm run build` が成功した。
- 意図しない変更が差分に含まれていない。
- 商品詳細ページの変更をコミットし、作業ブランチへ Push した。
